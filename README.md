# cert_creator — пакетная генерация PDF-сертификатов (Gramotadel)

Скрипт для Windows PowerShell отправляет запросы к API `gramotadel.express` (как форма на сайте) и для **каждого кода** из списка подставляет своё значение `mask_code`, остальные поля сертификата берутся из конфигурации. PDF сохраняются в папку (по умолчанию `out`).

## Требования

- Windows, **PowerShell 5.1** или новее (в т.ч. PowerShell 7).
- Доступ в интернет к `gramotadel.express` и `embed.gramotadel.express`.

## Быстрый старт

1. Скопируйте [config.example.json](config.example.json) в **`config.json`** и заполните поля (см. ниже). Реальные значения `secure` и `doc_id` удобно взять из браузера после успешной отправки формы на сайте.
2. Подготовьте **`codes.txt`**: один код сертификата на строку (можно ориентироваться на [codes.example.txt](codes.example.txt)).
3. В каталоге проекта выполните:

```powershell
.\Generate-Certificates.ps1
```

4. Откройте PDF в папке `out` (имя файла совпадает с кодом, недопустимые для Windows символы в имени заменяются на `_`).

### Параметры скрипта

| Параметр | По умолчанию | Описание |
|----------|----------------|----------|
| `-ConfigPath` | `config.json` | Путь к JSON-конфигу |
| `-CodesPath` | `codes.txt` | Файл со списком кодов |

Пример:

```powershell
.\Generate-Certificates.ps1 -ConfigPath .\config.json -CodesPath .\my_codes.txt
```

При любой ошибке по отдельным кодам скрипт продолжает работу; в конце выводится число успехов и неудач. Если **хотя бы один** код не удалось обработать, код выхода **1**.

## Формат `codes.txt`

- Одна строка — один код (`mask_code`).
- Пустые строки игнорируются.
- Строки, начинающиеся с `#`, считаются комментариями.

## Поля `config.json`

| Поле | Описание |
|------|----------|
| `mask_sum` | Сумма |
| `mask_date` | Срок действия (как на сайте, например `10.05.2026`) |
| `mask_city` | Город |
| `mask_adress` | Адрес |
| `mask_phone` | Телефон |
| `mask_valuta` | Валюта |
| `secure` | UUID из формы (см. ниже) |
| `doc_id` | UUID документа/шаблона |
| `form_id` | Идентификатор формы Tilda |
| `form_spec_comments` | Необязательный комментарий (`form-spec-comments` в запросе) |
| `origin`, `referer` | Заголовки запроса (как у страницы с формой, часто домен вашего сайта) |
| `user_agent` | User-Agent браузера (по желанию) |
| `api_url` | Необязательно, по умолчанию `https://gramotadel.express/api/v1/tilda-create/` |
| `output_dir` | Папка для PDF (по умолчанию `out`) |
| `request_delay_ms` | Пауза между запросами в миллисекундах (по умолчанию `300`) |

Откуда взять **`secure`** и **`doc_id`**: в браузере откройте инструменты разработчика (F12) → вкладка **Network** → отправьте форму на сайте → найдите запрос к `tilda-create` → в теле запроса (Form Data) скопируйте значения полей `secure` и `doc_id`.

## Как это устроено после ответа API

Ответ часто приходит HTML-виджетом с `data-url` на `embed.gramotadel.express`. Скрипт повторяет цепочку, близкую к браузеру: опрос `.../status/`, пока сертификат не готов, затем отправка формы виджета и загрузка PDF по адресу вида `https://embed.gramotadel.express/getfile/<id>/pdf/` с нужным заголовком `Referer`.

## Ограничения

- Значения `secure` / `doc_id` могут измениться, если форму на стороне Gramotadel/Tilda пересоберут — тогда снова снимите их из браузера.
- Массовая генерация должна соответствовать правилам и договору с Gramotadel и владельцем сайта формы.
