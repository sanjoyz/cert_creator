# cert_creator — пакетная генерация PDF-сертификатов (Gramotadel)

Скрипт для Windows PowerShell отправляет запросы к API `gramotadel.express` (как форма на сайте) и для **каждого кода** из списка подставляет своё значение `mask_code`, остальные поля сертификата берутся из конфигурации. PDF сохраняются в папку (по умолчанию `out`).

## Требования

- Windows, **PowerShell 5.1** или новее (в т.ч. PowerShell 7).
- Доступ в интернет к `gramotadel.express`, `embed.gramotadel.express`, при автоматическом получении токенов — к `auth.tildacdn.com` и странице с паролем Tilda (например `pixelquest.ru`).

## Быстрый старт

1. Скопируйте [config.example.json](config.example.json) в **`config.json`** и заполните поля (см. ниже).
2. **Токены `secure` и `doc_id`**: либо вставьте реальные UUID из браузера (Network → `tilda-create` → Form Data), либо оставьте плейсхолдеры `00000000-0000-0000-0000-000000000000` — скрипт сам пройдёт парольную страницу Tilda и подставит значения (нужен пароль, см. ниже).
3. Подготовьте **`codes.txt`**: один код сертификата на строку (можно ориентироваться на [codes.example.txt](codes.example.txt)), **кроме** режима `-Interactive`, где коды генерируются автоматически.
4. В каталоге проекта выполните:

```powershell
.\Generate-Certificates.ps1
```

5. Откройте PDF в папке `out` (имя файла совпадает с кодом, недопустимые для Windows символы в имени заменяются на `_`).

### Параметры скрипта

| Параметр | По умолчанию | Описание |
|----------|----------------|----------|
| `-ConfigPath` | `config.json` | Путь к JSON-конфигу |
| `-CodesPath` | `codes.txt` | Файл со списком кодов (не нужен при `-Interactive`) |
| `-Interactive` | выкл. | Спросить число кодов, опционально `mask_sum` / `mask_date`, сгенерировать случайные коды; опционально открыть `config.json` в блокноте |
| `-FetchTokens` | выкл. | Принудительно заново получить `secure`/`doc_id` через Tilda (даже если в конфиге уже не плейсхолдеры) |

Примеры:

```powershell
.\Generate-Certificates.ps1 -ConfigPath .\config.json -CodesPath .\my_codes.txt
.\Generate-Certificates.ps1 -Interactive
.\Generate-Certificates.ps1 -FetchTokens
```

При любой ошибке по отдельным кодам скрипт продолжает работу; в конце выводится число успехов и неудач. Если **хотя бы один** код не удалось обработать, код выхода **1**.

### Пароль страницы Tilda (автополучение токенов)

**Не храните пароль в репозитории.** Задайте переменную окружения пользователя или процесса:

- `CERT_TILDA_PASSWORD` — обычная строка с паролем.

Если переменная не задана, при необходимости получить токены скрипт запросит пароль через `Read-Host -AsSecureString`.

Запрос к Tilda выполняется **один раз** перед циклом по кодам (не на каждый сертификат).

## Формат `codes.txt`

- Одна строка — один код (`mask_code`).
- Пустые строки игнорируются.
- Строки, начинающиеся с `#`, считаются комментариями.

## Поля `config.json`

| Поле | Описание |
|------|----------|
| `mask_sum` | Сумма |
| `mask_date` | Срок действия (как на сайте, например `10.05.2026`) |
| `mask_city` | Город |
| `mask_adress` | Адрес |
| `mask_phone` | Телефон |
| `mask_valuta` | Валюта |
| `secure` | UUID из скрытого поля формы |
| `doc_id` | UUID документа/шаблона |
| `form_id` | Идентификатор формы Tilda |
| `form_spec_comments` | Необязательный комментарий (`form-spec-comments` в запросе) |
| `origin`, `referer` | Заголовки запроса (как у страницы с формой, часто домен вашего сайта) |
| `user_agent` | User-Agent браузера (по желанию) |
| `api_url` | Необязательно, по умолчанию `https://gramotadel.express/api/v1/tilda-create/` |
| `output_dir` | Папка для PDF (по умолчанию `out`) |
| `request_delay_ms` | Пауза между запросами в миллисекундах (по умолчанию `300`) |
| `tilda_page_url` | URL страницы с паролем Tilda (по умолчанию `https://pixelquest.ru/cert`) |
| `tilda_project_id` | `data-tilda-project-id` со страницы (пример в шаблоне) |
| `tilda_page_id` | `data-tilda-page-id` со страницы (пример в шаблоне) |
| `certificate_form_id` | Атрибут `id` формы в разметке Tilda, из которой читаются `secure` и `doc_id` (например `form777631482`) |

**Откуда взять `tilda_project_id` / `tilda_page_id`:** откройте страницу с паролем в браузере → просмотр HTML → атрибуты `data-tilda-project-id` и `data-tilda-page-id` у корневого блока Tilda.

**Альтернатива ручным токенам:** в DevTools → Network отправьте форму на сайте → запрос `tilda-create` → в Form Data скопируйте `secure` и `doc_id`.

## Как устроено после ответа API

Ответ часто приходит HTML-виджетом с `data-url` на `embed.gramotadel.express`. Скрипт повторяет цепочку, близкую к браузеру: опрос `.../status/`, пока сертификат не готов, затем отправка формы виджета и загрузка PDF по адресу вида `https://embed.gramotadel.express/getfile/<id>/pdf/` с нужным заголовком `Referer`.

## Автоматическое получение `secure` / `doc_id` (кратко)

1. Загружается шаблон с `auth.tildacdn.com` (session и шаблонный csrf).
2. Два POST на `https://auth.tildacdn.com/api/accesspage`: сначала с пустым паролем для `csrf_val`, затем с реальным паролем.
3. Из JSON-ответа в поле `content` парсится HTML нужной формы (`certificate_form_id`), из неё извлекаются скрытые поля `secure` и `doc_id`.

## Ограничения

- Значения `secure` / `doc_id` могут измениться, если форму на стороне Gramotadel/Tilda пересоберут — тогда снова получите токены (флаг `-FetchTokens` или плейсхолдеры в конфиге).
- Массовая генерация должна соответствовать правилам и договору с Gramotadel и владельцем сайта формы.
